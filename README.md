# Russian-Text-Classification-Pipeline
Сбор текстовых данных из открытых источников и построение пайплайна машинного обучения для бинарной тематической классификации (Кулинария vs Спорт). Проект включает в себя парсинг данных (Telegram, VK API, веб-скрапинг), лингвистическую предобработку, а также системное сравнение классических ML-алгоритмов (на признаках TF-IDF/CountVectorizer) и современной Transformer-модели RuBERT.

## О проекте

Этот проект представляет собой end-to-end исследование в области обработки естественного языка (NLP). Анализ выполнен на самостоятельно собранном корпусе данных, содержащем публикации из социальных сетей и тематических сайтов.

Основная цель работы — разработать воспроизводимый пайплайн классификации и сравнить эффективность традиционных подходов машинного обучения на разреженных матрицах с современными нейросетевыми архитектурами (Transformers) на примере задачи с семантически удаленными классами.

## Структура проекта

*   `NLP_Classification.ipynb` — Jupyter Notebook, содержащий весь код для парсинга, очистки данных, построения и оценки моделей.
*   `data/` — директория с исходными датасетами, собранными из Telegram, VK и сайтов.
*   `requirements.txt` — список необходимых библиотек для воспроизведения окружения.
*   `README.md` — данный файл с описанием проекта.

## Этапы работы

1.  **Сбор данных (Data Mining)**:
    *   Реализован асинхронный сбор сообщений из Telegram-каналов с помощью библиотеки Pyrogram.
    *   Выполнена выгрузка постов из открытых тематических сообществ ВКонтакте через VK API.
    *   Проведен веб-скрапинг кулинарных рецептов и спортивных новостей с использованием BeautifulSoup (с нормализацией дат и текста).
    *   Итоговый сбалансированный корпус составил 4006 текстов.

2.  **Лингвистическая предобработка**:
    *   Разработано два независимых пайплайна в виде scikit-learn трансформеров для предотвращения утечки данных (data leakage) при кросс-валидации.
    *   Для классических моделей: применена агрессивная очистка регулярными выражениями, удаление стоп-слов и лемматизация с помощью PyMorphy3.
    *   Для RuBERT: использована мягкая очистка от технического шума (URL, HTML) с сохранением регистра, пунктуации и естественной морфологии.

3.  **Системный перебор классических моделей (Baselines)**:
    *   Проведено тестирование 105 конфигураций гиперпараметров в рамках 5-fold кросс-валидации.
    *   Сравнивались различные методы векторизации (Bag of Words, TF-IDF, N-grams) и алгоритмы классификации (Logistic Regression, Linear SVC, SGD, Ridge, Naive Bayes, Random Forest, KNN).

4.  **Дообучение Transformer-модели**:
    *   Выполнен fine-tuning легковесной языковой модели `cointegrated/rubert-tiny2` с использованием PyTorch и Hugging Face Transformers.

## Результаты

Финальные результаты показали, что обе парадигмы машинного обучения отлично справляются с задачей ввиду сильной тематической удаленности классов (анализ облаков слов подтвердил практически непересекающийся словарный состав).

*   **Классические методы**: Лучшая конфигурация (Лемматизация + Bag of Words + SGD Classifier) достигла метрики **F1-Macro = 0.9998**. Модель отличается высокой скоростью обучения и инференса.
*   **RuBERT**: После 3 эпох fine-tuning модель достигла метрики **F1-Macro = 1.0000**.

Ключевые выводы:
1. На задачах с хорошо разделимыми классами и выраженным набором ключевых слов классические линейные модели на разреженных признаках практически не уступают тяжелым нейросетевым архитектурам.
2. Лемматизация текстов стабильно повышает качество классификации для всех базовых алгоритмов.
3. Использование сложных Transformer-моделей (таких как BERT) для подобных задач избыточно, так как линейный классификатор решает задачу с почти идеальной точностью при минимальных вычислительных затратах. Преимущество BERT проявилось бы на более сложных датасетах с близкими тематиками или скрытым контекстом.
